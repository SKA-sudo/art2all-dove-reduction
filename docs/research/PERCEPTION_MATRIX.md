# PERCEPTION_MATRIX.md

## Perception Matrix Methodology

### Purpose

The Perception Matrix defines the standardized scientific observation methodology used by the Reduction Lab.

Its purpose is to ensure that every perception experiment is documented using the same workflow and the same observation criteria. This enables objective comparison between different reduction experiments.

The Perception Matrix does **not** define perception rules and does **not** evaluate experimental results. It defines only the methodology used to collect observations.

---


## Research Phases

| Phase              | Frage                                      | Erlaubt                            | Nicht erlaubt                      |
| ------------------ | ------------------------------------------ | ---------------------------------- | ---------------------------------- |
| **1. Observation** | Was sehe ich?                              | reine Beschreibung der Wahrnehmung | Erklärungen, Anatomie, Algorithmen |
| **2. Hypothesis**  | Warum könnte das Gehirn das so wahrnehmen? | Vermutungen                        | Implementierung                    |
| **3. Reduction**   | Welche Information entferne ich?           | genau **eine** Änderung            | mehrere Änderungen gleichzeitig    |
| **4. Validation**  | Hat sich die Wahrnehmung verändert?        | Beobachtung                        | neue Hypothesen                    |


Observation → Perception → Hypothesis → Reduction → Validation → Perception Rule

Every perception experiment follows four strictly separated phases.

### Phase 1 – Observation



Describe only what is directly perceived.

Allowed:

* Immediate visual perception
* Neutral description
* Observable facts

Not allowed:

* Explanations
* Hypotheses
* Anatomical reasoning
* Algorithms
* Implementation ideas

---

### Phase 2 – Hypothesis

Formulate possible explanations for the observed perception.

Allowed:

* Assumptions
* Perceptual interpretations
* Scientific reasoning

Not allowed:

* Implementation
* Validation
* Conclusions presented as facts

---

### Phase 3 – Reduction

Modify exactly one visual property at a time.

Rules:

* Only one change per experiment.
* All other properties remain unchanged.
* The reduction must be reproducible.

---

### Phase 4 – Validation

Observe the modified result using the same procedure as in Phase 1.

Questions:

* Has the perception changed?
* Which perception remained stable?
* Which perception disappeared?
* Does the hypothesis still explain the observation?

Only after successful validation may a perception rule be formulated.



# 1. Observation Workflow

Every future experiment follows the identical workflow.

```
Image
    ↓
Observation
    ↓
Documentation
    ↓
Semantic Analysis
    ↓
Cross Validation
    ↓
Hypothesis
    ↓
Validation
    ↓
Perception Rule
```

## Step 1 — Image

A single visual representation is selected for observation.

No interpretation or analysis is performed at this stage.

---

## Step 2 — Observation

The observer records the immediate visual perception.

Only direct observations are documented.

No explanations or conclusions are made.

---

## Step 3 — Documentation

All observations are recorded using the standardized experiment template.

Every experiment uses the same structure.

No fields may be omitted.

---

## Step 4 — Semantic Analysis

The recorded observations are organized according to the predefined semantic observation categories.

No perception rules are derived during this step.

---

## Step 5 — Cross Validation

Observations from multiple experiments or observers are compared to determine consistency.

Only repeatable observations continue to the next stage.

---

## Step 6 — Hypothesis

A possible explanation for the observed perception is formulated.

The hypothesis remains provisional until validated.

---

## Step 7 — Validation

The hypothesis is tested through additional reduction experiments.

Only reproducible results are considered valid.

---

## Step 8 — Perception Rule

If repeated validation confirms the hypothesis, it may be documented as a validated perception rule.

Perception rules are maintained separately from this methodology document.


# 2. Experiment Template

Every perception experiment is documented using the following template.

The template is mandatory for every experiment performed in the Reduction Lab.

---

## Experiment Information

**Experiment ID**

Unique identifier of the experiment.

**Date**

Date of observation.

**Observer**

Person performing the observation.

**Image**

Reference to the observed image.

**Reduction Stage**

Version or reduction level of the image.

---

## Initial Observation

**First Perception (0–2 s)**

The first spontaneous interpretation.

**Confidence**

How certain the observer is about the initial perception.

**Recognition Time**

Approximate time until recognition.

**Alternative Perceptions**

Any competing interpretations that appeared.

---

## Semantic Observation

The observation is documented using the predefined semantic categories.

No conclusions are drawn during this step.

Only observations are recorded.

---

## Comments

Free observations that are not covered by the semantic categories.

---

## Cross Validation

Space reserved for later comparison with additional observers or experiments.

No validation is performed during the initial observation.

---

## Hypothesis

Reserved.

No hypothesis is created during documentation.

---

## Validation

Reserved.

Validation is performed only after sufficient comparable experiments exist.

---

## Perception Rule

Reserved.

This field remains empty until a hypothesis has been validated through repeated experiments.

Validated perception rules are maintained separately from the experiment documentation.

# 3. Semantic Observation Categories

Semantic observation categories provide a standardized vocabulary for describing visual perception.

They are descriptive only.

They do not explain why a perception occurs and they do not define perception rules.

Multiple categories may apply simultaneously.

---

## Recognition

Describes what is perceived.

Examples:

* First perception
* Alternative perception
* Semantic stability

---

## Spatial Organization

Describes the spatial arrangement of visual elements.

Examples:

* Center
* Dominant axis
* Secondary axis
* Balance
* Symmetry
* Asymmetry
* Direction

---

## Structural Organization

Describes how visual elements relate to each other.

Examples:

* Connection
* Separation
* Grouping
* Hierarchy
* Continuity
* Branching
* Layering

---

## Geometric Characteristics

Describes geometric properties without interpretation.

Examples:

* Curvature
* Straightness
* Expansion
* Contraction
* Open form
* Closed form

---

## Visual Dynamics

Describes perceived movement or visual flow.

Examples:

* Movement
* Rhythm
* Repetition
* Orientation
* Visual flow

---

## Semantic Relationships

Describes meaningful relationships between visual structures.

Examples:

* Common origin
* Shared direction
* Structural dependency
* Visual dominance
* Local relationship
* Global relationship

---

## Observation Notes

Additional observations that are not represented by the predefined categories may be documented here.

These notes are considered descriptive observations only and do not constitute perception rules.

# 4. Methodology Review

The purpose of this review is to verify that the Perception Matrix provides a complete and stable methodology for future perception experiments.

This review evaluates the methodology itself.

It does not evaluate experimental results.

---

## Review Checklist

The methodology is considered complete if all of the following statements are true.

### Observation Workflow

* Every experiment follows the same workflow.
* The workflow can be repeated without modification.
* The workflow is independent of the observed image.

### Experiment Template

* Every required observation can be documented.
* The template contains all mandatory information.
* No experiment requires additional documentation fields.

### Semantic Observation Categories

* Every relevant observation can be assigned to one or more categories.
* Categories describe observations only.
* Categories do not contain explanations or perception rules.
* The category set is internally consistent.

### Repeatability

* Different observers can use the methodology without additional instructions.
* Different experiments remain directly comparable.
* Documentation is independent of the expected outcome.

### Scientific Neutrality

* Observations are recorded before interpretation.
* Hypotheses are created only after observations have been documented.
* Validation is performed only through repeated experiments.
* Perception rules are created only after successful validation.

---

## Completion Criteria

The methodology is complete when:

* the observation workflow is stable,
* the experiment template is complete,
* the semantic categories are sufficient,
* future perception experiments can be documented without modifying this document.

Once these criteria are satisfied, the methodology becomes the standard documentation framework for all experiments performed in the Reduction Lab.
